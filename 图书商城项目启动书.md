# 项目启动书：图书商城平台

## 项目概况

项目名称：图书商城平台

开发模式：前后端分离

技术栈：后端 JAVA，前端 Vue.js

主要受众：大众

成员：张锦成 刘骏毅 黄栋扬 黄均鹏

## 背景与意义

### 现状与问题

**校园购书渠道分散**：学生在购买图书时，往往需要在校园书店、校外实体书城、线上电商平台之间来回切换，缺乏一个专注校园场景的图书购买入口。

**图书信息获取滞后**：学生难以实时了解校园书店的新书上架、库存情况或促销活动，常常错过心仪图书或跑空书店。

**校园书店运营方式传统**：校内书店多依赖人工管理，图书信息更新慢，缺乏数字化手段触达学生读者群体。

**阅读需求与购书行为脱节**：学生的阅读兴趣、课程推荐书目与购书渠道之间缺乏有效连接，购书行为多为被动搜索，缺少个性化推荐。

### 项目意义

**打造专注校园场景的图书商城平台**：整合校内及周边图书资源，为学生提供便捷的线上购书入口，提升购书体验。

**优化图书信息展示与查询**：通过线上图书展示、库存查询、新书推荐等功能，让学生随时随地掌握图书动态，减少跑空。

**赋能校园书店数字化升级**：为校园书店提供便捷的后台管理系统，实现图书信息、库存、促销活动的实时更新与管理。

**构建校园阅读文化生态**：以图书商城为核心，逐步拓展至阅读推荐、书评交流、课程用书预订等功能，培育校园阅读氛围。

**推动校园文化服务数字化转型**：作为校园文化建设的重要组成部分，助力校园图书服务向数字化、智能化发展。

## 项目目标

**构建一站式校园图书购物平台**：整合校园及周边图书资源，实现在线图书浏览、库存查询、线上下单等功能，为学生提供便捷的购书体验。

**实现智能图书检索与推荐**：支持按书名、作者、分类等关键字搜索，并提供新书推荐、热销榜单、个性化书单等功能，帮助学生发现好书。

**提升校园图书服务数字化水平**：通过数据统计分析，为校园书店提供经营决策支持，推动校园图书服务向智能化转型。

## 系统功能范围

### 用户端（前台）

| 功能模块 | 功能描述 |
| :--- | :--- |
| 登录注册 | 支持用户注册登录，是购物、评价、收藏等功能的前提。 |
| 首页浏览 | 查看店长推荐、新书速递、图书销售Top5。 |
| 分类浏览 | 按分类浏览图书，查看分类列表及对应图书。 |
| 图书搜索 | 支持按书名、作者等关键字模糊搜索图书。 |
| 图书详情 | 查看图书封面、作者、价格、简介及用户评价。 |
| 购物车 | 将图书加入/移出购物车，管理选购商品并进行结算。 |
| 订单管理 | 查看订单状态、确认收货、对已完成订单进行评价。 |
| 支付功能 | 使用账户余额进行订单支付。 |
| 评价功能 | 对已购图书进行评分和评论。 |
| 账户充值 | 用户可对账户进行余额充值。 |
| 个人中心 | 查看和修改个人资料、重置密码、管理收货地址。 |

### 管理端（后台）

| 功能模块 | 功能描述 |
| :--- | :--- |
| 登录功能 | 管理员通过账号密码登录系统。 |
| 数据统计 | 首页展示用户总数、图书总数、订单总数、分类总数等关键数据。 |
| 图书信息管理 | 支持按分类、书名、作者等条件查询图书；支持增、删、改、查；可设置"今日推荐"；管理封面、价格、描述等信息。 |
| 图书分类管理 | 支持分类名称、排序设置；支持增、删、改、查分类信息。 |
| 用户购物车管理 | 查看用户购物车记录，支持按书名、用户名查询；显示图书信息、数量、用户名等；支持增、删、改。 |
| 订单管理 | 查看订单列表，支持按订单号、状态、用户查询；显示订单号、总价、收货信息、状态等；支持发货、查看详情、增、改操作。 |
| 收货地址管理 | 查看用户收货地址，支持按收货人、用户名查询；显示收货人、电话、地址、用户名；支持增、删、改。 |
| 图书评价管理 | 查看图书评价列表，支持按图书名称、评价用户查询；显示评分、内容、用户、时间；支持增、删、改。 |

## 业务约束与非功能需求

**业务规则**：同一用户对同一订单不可重复评价，订单完成后才可提交评价；用户下单时需实时校验图书库存，库存不足时无法提交订单；订单支付时限为30分钟，超时未支付系统自动取消订单并释放库存。

**数据维护**：系统需具备专业的管理后台，确保图书信息、订单数据、用户信息的准确性与完整性，以保障平台运营的专业性和用户信任；图书分类、商品信息等基础数据需由管理员定期审核更新，确保展示内容的时效性与准确性。

**核心指标**：优化用户购书流程，提升订单处理效率，减少用户从浏览到完成购买的操作步骤与等待时间；缓解线下书店排队拥挤情况，通过线上下单、到店自提或配送服务分流线下客流。
